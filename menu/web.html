Web S2

// Résumé du cours

Ce cours couvre les fondements du développement web dynamique avec PHP et MySQL. Il présente d'abord la distinction entre pages statiques et dynamiques, puis la syntaxe PHP (variables, types, opérateurs, tableaux, structures de contrôle, fonctions), et enfin l'accès aux bases de données via PDO avec les requêtes préparées.

1 — Pages statiques vs dynamiques
✕  Pages statiques — limites
  • Fichier HTML fixe, même réponse pour tous.
  • Aucun calcul ni traitement conditionnel possible.
  • Mises à jour manuelles fastidieuses.
  • Pas d'interfaçage avec une base de données.
✓  Pages dynamiques — avantages
  • Script côté serveur interprété à chaque requête.
  • Contenu généré selon le demandeur ou les données.
  • Compatibles avec tous les navigateurs et OS.
  • Code source masqué : le client ne voit que le HTML.
Langage côté serveur : programme stocké sur le serveur qui parcourt un fichier source, en modifie le contenu (ex : injection de données MySQL), puis renvoie le HTML résultant au navigateur. PHP, combiné à Apache et MySQL, forme la solution LAMP — gratuite, robuste et stable.
2 — PHP : syntaxe et bases
Historique

Créé en 1994 par Rasmus Lerdorf (Personal Home Pages). V3 en 1998, PHP4 en 1999. En 2001, 5 millions de domaines l'utilisent.

Caractéristiques
  • Exécuté côté serveur — indépendant du client.
  • Syntaxe inspirée de C, Perl et Java.
  • Gratuit, open-source (licence GNU GPL).
  • S'intègre directement dans le HTML.
Règles syntaxiques
  • Balise ouvrante : <?php … ?>
  • Toute instruction se termine par ;
  • Sensible à la casse (sauf les fonctions).
  • Commentaires : // ... ou /* ... */

Variables & Types

▼

Les variables sont préfixées par $ et non typées : elles peuvent changer de type en cours de script. Les variables de formulaire sont toujours de type string.

// Variables et portée 
<?php
$annee = 1970;           // integer
$nom   = "Jean-Michel";  // string
echo "Bonjour $nom, né en $annee";

// Variable globale dans une fonction
$var = 100;
function test() {
    global $var;
    return $var;
}

// Variable statique — conserve sa valeur entre les appels
function cumul($prix) {
    static $total = 0;
    $total += $prix;
    return $total;
}
?>
Types disponibles
integer

Nombres entiers.

double / float

Nombres à virgule flottante.

string

Chaînes de caractères.

array

Tableaux indexés ou associatifs.

bool

Booléen vrai / faux.

object

Instances de classes PHP.

Transtypage : settype($var, "double") convertit le type. Le cast explicite fonctionne aussi : (int)$var, (string)$var, etc.

Tableaux

▼
Tableau simple

Index numérique dès 0. Créé avec array() ou affectation directe.

Tableau associatif

Clés textuelles (clé => valeur). Accès via $tab["clé"].

Tableau multidim.

Tableaux imbriqués. Accès via $tab[i][j].

// Tableau simple + parcours 
$jour = array("Dim", "Lun", "Mar", "Mer");
echo $jour[2]; // → "Mar"

// Boucle for
for ($i = 0; $i < count($jour); $i++) {
    echo $jour[$i];
}

// Boucle foreach (recommandée)
foreach ($jour as $JJ) {
    echo $JJ . "<br>";
}
// Tableau associatif 
$semaine = array("Dimanche" => 1, "Lundi" => 2);
echo $semaine["Lundi"]; // → 2

foreach ($semaine as $sJJ => $nJJ) {
    echo "Jour n°$nJJ : $sJJ<br>";
}

Structures de contrôle

▼
Branchements conditionnels
// if / else 
if ($valeur > 1) {
    echo "supérieure à 1";
} elseif ($valeur == 1) {
    echo "égale à 1";
} else {
    echo "inférieure à 1";
}
// switch 
switch ($variable) {
    case "valeur1": /* instructions */ break;
    case "valeur2": /* instructions */ break;
    default:       /* instructions */ break;
}
Boucles
// while 
$i = 1; $somme = 0;
while ($i <= 10) {
    $somme += $i;
    $i++;
}
echo "Somme 1 à 10 = $somme";
// for 
for ($i = 1; $i <= 10; $i++) {
    $somme += $i;
}
// foreach — recommandé pour les tableaux 
// Simple
foreach ($tableau as $valeur) {
    echo $valeur;
}

// Avec clé (associatif)
foreach ($tableau as $cle => $valeur) {
    echo "$cle : $valeur";
}

Fonctions

▼
Déclaration

Mot-clé function, paramètres entre parenthèses, retour via return.

Passage par référence

Préfixer le paramètre de & pour modifier la variable d'origine directement.

Args. variables

func_num_args() — nb d'args. func_get_arg($i) — valeur du i-ème arg.

Inclusions

require() insère le fichier une fois. include() l'évalue à chaque appel (utilisable en boucle).

// Syntaxe + passage par référence 
function aire($l, $h) {
    return $l * $h;
}
echo aire(5, 3); // → 15

// Par référence : modifie la variable d'origine
function doubler(&$val) { $val *= 2; }
$x = 5;
doubler($x);
echo $x; // → 10

3 — PHP & MySQL : accès aux bases de données
// Principe

PHP s'interface avec MySQL pour générer des pages dynamiques depuis des données stockées en base. L'extension PDO (PHP Data Objects) fournit une interface orientée objet standardisée pour établir des connexions sécurisées et exécuter des requêtes SQL.

4 paramètres requis
  • Hôte — adresse IP ou localhost
  • Base — nom de la BDD
  • Identifiant — login MySQL
  • Mot de passe — à ne jamais exposer
Sécurité

Toujours encapsuler la connexion dans un try/catch. Sans cela, PHP peut afficher le mot de passe en cas d'erreur.

Requêtes préparées

Utilisées dès qu'un paramètre est dynamique (formulaire, etc.). Les marqueurs nommés (:nom) protègent contre les injections SQL.

Connexion à la base de données

▼
// Connexion sécurisée avec try/catch 
<?php
try {
    $mysqlConnection = new PDO(
        'mysql:host=localhost; dbname=ma_base; charset=utf8',
        'utilisateur',
        'motDePasse'
    );
} catch (Exception $e) {
    die('Erreur : ' . $e->getMessage());
}
?>
⚠ Sans try/catch, PHP risque d'afficher la ligne entière provoquant l'erreur — ce qui expose le mot de passe dans le navigateur ou les logs serveur.

Récupérer des données — SELECT simple

▼
// 3 étapes : prepare → execute → fetchAll 
<?php
// 1 — Déclaration de la requête
$requete = $mysqlConnection->prepare('SELECT * FROM nom_table');

// 2 — Exécution
$requete->execute();

// 3 — Récupération en tableau
$resultat = $requete->fetchAll();

// Parcours des résultats
foreach ($resultat as $ligne) {
    echo $ligne['nom_colonne'];
}
?>

Requêtes préparées — paramètres dynamiques

▼

Quand la clause WHERE dépend d'une variable (ex : saisie d'un formulaire), on utilise des paramètres nommés pour sécuriser l'exécution.

// Requête avec paramètres :nom et :prenom 
<?php
// Définition de la requête paramétrée
$sql = 'SELECT * FROM Utilisateur 
        WHERE nomUser = :nom AND prenomUser = :prenom';

// Préparation
$requete = $mysqlConnection->prepare($sql);

// Exécution avec les valeurs réelles
$requete->execute([
    'nom'    => 'Durand',
    'prenom' => 'Martin',
]);

$resultat = $requete->fetchAll();
?>
Lifecycle des requêtes SQL avec PDO
ÉtapeMéthodeRôle
1. Préparationprepare()Définit la structure SQL avec des espaces réservés (:nom).
2. Exécutionexecute()Envoie la requête au serveur avec les valeurs réelles des paramètres.
3. RécupérationfetchAll()Transforme le jeu de résultats en tableau PHP exploitable.
Vocabulaire technique
PDO

PHP Data Objects — interface orientée objet standard pour accéder à diverses bases de données relationnelles depuis PHP.

Requête préparée

Instruction SQL template avec paramètres nommés (:nom). Sépare la structure SQL des données, protège contre les injections.

fetchAll()

Retourne l'intégralité du jeu de résultats d'une requête sous forme de tableau PHP associatif.

Script côté serveur

Programme exécuté par le serveur avant d'envoyer le HTML au navigateur. Code source masqué, compatible tous navigateurs.

try / catch

Structure de gestion des exceptions. Indispensable autour de la connexion PDO pour éviter l'exposition d'informations sensibles.

foreach

Boucle de parcours de tableau. Recommandée pour itérer sur les résultats SQL retournés par fetchAll().

static

Mot-clé PHP permettant à une variable locale de conserver sa valeur entre deux appels successifs de la même fonction.

include / require

include() insère un fichier à chaque appel (utilisable en boucle). require() produit une erreur fatale si le fichier est absent.
